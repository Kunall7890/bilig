Who needs this

Teams that still audit rules in cells.

Bilig is for the middle nobody loves: the logic belongs in a workbook because people trust the cells, but production needs a service, queue, job, or agent tool to run it without a spreadsheet session.

primary runtime Workbook-shaped business logic, service-owned execution.

Put pricing, payout, quote, finance, import-validation, and workflow rules behind a TypeScript API while keeping the workbook structure reviewable.

Market gap

Not another spreadsheet UI. Not just a formula engine.

File libraries move workbook bytes. Formula engines calculate formulas. Hosted spreadsheets need networked app state. Bilig targets the missing runtime layer for backend teams that need workbook-shaped logic under test.

01 / model

Keep the business rule readable.

Represent inputs, formulas, and outputs as sheets instead of burying every rule in custom service code.

02 / run

Let the service change the inputs.

Write values through a narrow API, recalculate dependent formulas, and read the cells that determine the decision.

03 / save

Persist the workbook state.

Save WorkPaper JSON for route handlers, queue jobs, tests, agent tools, and audit trails.

04 / bridge

Import or export XLSX when files matter.

Use the XLSX subpath and recalculation packages when the workflow still needs Excel-compatible files.

Quickstart

Use the runtime directly.

Do the real loop first: install the runtime, write an input cell, read the calculated value, save JSON, restore it, and read again.

Blank Node project

terminal Node.js 22+ 
mkdir workpaper-service
cd workpaper-service
npm init -y
npm pkg set type=module
npm install @bilig/headless
npm install -D tsx typescript @types/node
npx tsx quickstart.ts

Prefer scaffolding? Run npm create @bilig/workpaper@latest pricing-workpaper. For a route-shaped version, use examples/serverless-workpaper-api. For an existing Excel file, use examples/xlsx-recalculation-node.

quickstart.ts TypeScript 
import {
  WorkPaper,
  createWorkPaperFromDocument,
  exportWorkPaperDocument,
  parseWorkPaperDocument,
  serializeWorkPaperDocument,
} from "@bilig/headless";

const paper = WorkPaper.buildFromSheets({
  Inputs: [["Metric", "Value"], ["Customers", 20], ["ARR", 1200]],
  Summary: [["Metric", "Value"], ["Revenue", "=Inputs!B2*Inputs!B3"]],
});

const inputs = paper.getSheetId("Inputs");
const summary = paper.getSheetId("Summary");
if (inputs === undefined || summary === undefined) throw new Error("missing sheet");

paper.setCellContents({ sheet: inputs, row: 1, col: 1 }, 40);
const revenue = paper.getCellDisplayValue({ sheet: summary, row: 1, col: 1 });
const saved = serializeWorkPaperDocument(exportWorkPaperDocument(paper));
const restored = createWorkPaperFromDocument(parseWorkPaperDocument(saved));

console.log({ revenue, restoredSheets: restored.getSheetNames() });
