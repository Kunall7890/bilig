# Deslop Public Surfaces Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove low-value, repetitive, stale, or generic public content from Bilig while keeping runnable WorkPaper proof, current limits, and adoption paths easy to find.

**Architecture:** Treat slop as a content-quality bug: every public entry point must have one job, one current proof path, and one clear boundary. Edit canonical Markdown first, regenerate derived discovery files through repo scripts, and validate with docs checks before committing.

**Tech Stack:** Markdown, TypeScript discovery checks, `pnpm`, GitHub CLI, npm registry readback.

---

## Cleanup Standard

Use these external anchors for the working definition:

- AI slop is low-quality AI-generated content, usually produced at scale without enough useful substance: <https://en.wikipedia.org/wiki/AI_slop>.
- Google treats scaled low-value content as spam whether generated by humans, automation, or a mix: <https://developers.google.com/search/docs/essentials/spam-policies#scaled-content-abuse>.
- Helpful content should be made for readers first and should demonstrate first-hand usefulness: <https://developers.google.com/search/docs/fundamentals/creating-helpful-content>.

For this repo, delete or rewrite content when it is:

- repeated in multiple entry points without adding a new decision;
- stale against `origin/main`, npm latest, GitHub issue state, or live package proofs;
- XLSX-first when the page is supposed to explain WorkPaper runtime proof;
- a directory of links where a reader needs one next command;
- a broad claim without a runnable evaluator, benchmark artifact, limitation, or issue link.

Do not remove:

- runnable evaluator commands;
- current proof transcripts;
- compatibility limits;
- generated discovery files that are regenerated by scripts;
- saved-file docs when the page is explicitly about import/export or XLSX recalculation.
