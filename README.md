Bilig is an agent-first workbook platform.

It is not trying to be a prettier spreadsheet grid. Excel, Sheets, and
LibreOffice were built for humans looking at cells. Bilig is built for agents,
services, tests, and product code that need workbook logic as a contract:
inspectable, executable, and verifiable without driving a UI.

The rule is direct: an agent must love this library. The public API should be
simple, generic, predictable, inspectable, verifiable, and free of hardcoded
business models or human spreadsheet UI assumptions.

Project site: <https://proompteng.github.io/bilig/>

## What To Use

| Need | Use | Why |
| --- | --- | --- |
| Define a workbook model an agent can inspect and run | [`@bilig/workbook`](packages/workbook/README.md) | Consumer-defined `find`, `checks`, and `actions` that produce portable workbook intent. |
| Run workbook formulas in Node.js | [`@bilig/workpaper`](packages/workpaper/README.md) | Headless formula workbooks for services, queues, tests, and agent tools. |
| Edit inputs and recalculate XLSX bytes | [`@bilig/xlsx-formula-recalc`](packages/xlsx-formula-recalc/README.md) | File-level recalculation without opening Excel, LibreOffice, Google Sheets, or a browser. |
| Keep SheetJS or ExcelJS at the file boundary | [`@bilig/sheetjs-formula-recalc`](packages/sheetjs-formula-recalc/README.md) / [`@bilig/exceljs-formula-recalc`](packages/exceljs-formula-recalc/README.md) | Recalculate through the workbook library you already use. |
| Work on the Bilig app | [`apps/bilig`](apps/bilig) and [`apps/web`](apps/web) | Fullstack runtime plus browser workbook shell. |

## Agent-First Workbook Models

`@bilig/workbook` is the public package for generic, consumer-defined workbook
models. Bilig does not ship revenue models, prepaid models, reporting models, or
other business templates. Consumers define their own model; Bilig provides the
stable workbook language, formula helpers, checks, plan validation, and runtime
handoff.

```ts
import { defineModel, formula } from "@bilig/workbook";

export const model = defineModel({
  name: "custom-calculation",

  find(workbook) {
    const table = workbook.findTable({
      headers: ["Base", "Rate", "Result"],
    });

    return {
      table,
      base: table.column("Base"),
      rate: table.column("Rate"),
      result: table.column("Result"),
    };
  },

  checks({ refs, workbook }) {
    return [
      workbook.check.exists(refs.table),
      workbook.check.noFormulaErrors(refs.result),
    ];
  },

  actions: {
    calculate({ refs, workbook }) {
      workbook.writeFormula(
        refs.result,
        formula.multiply(refs.base, refs.rate),
      );
      workbook.check.noFormulaErrors(refs.result);
    },
  },
});
```

Agent flow:

1. Inspect the model with `describeModel(model)`.
2. Plan with `planWorkbookAction(model, actionName, input)`.
3. Verify with `verifyPlan(plan)`.
4. Execute with `runWorkbookAction` or a runtime adapter.
5. Accept success only when `WorkbookRunResult.status === "done"` and runtime
   checks come back as proof.

Formula ownership stays clean:

```text
@bilig/workbook creates formula expressions
@bilig/formula parses and normalizes formula language
@bilig/core calculates formulas
```

## Headless WorkPaper Runtime

Use `@bilig/workpaper` when a Node.js service, queue worker, route handler, test,
or agent tool needs to write inputs, recalculate formulas, read values, and save
workbook state without opening a spreadsheet UI.

```sh
npm create @bilig/workpaper@latest pricing-agent -- --agent
cd pricing-agent
npm run agent:verify
```

Starter projects are generated by the published `@bilig/create-workpaper`
package.

Minimal TypeScript shape:

```ts
import { WorkPaper } from "@bilig/workpaper";

const workbook = WorkPaper.buildFromSheets({
  Inputs: [
    ["Metric", "Value"],
    ["Units", 40],
    ["Price", 1200],
  ],
  Summary: [
    ["Metric", "Value"],
    ["Revenue", "=Inputs!B2*Inputs!B3"],
  ],
});

const inputs = workbook.getSheetId("Inputs");
const summary = workbook.getSheetId("Summary");

if (inputs === undefined || summary === undefined) {
  throw new Error("Expected sheets to exist");
}

workbook.setCellContents({ sheet: inputs, row: 1, col: 1 }, 48);

console.log(workbook.getCellValue({ sheet: summary, row: 1, col: 1 }));
workbook.dispose();
```

## TypeScript API Shape

The WorkPaper runtime shape is intentionally small: build sheets, write an
input, read the calculated value, and serialize state.

```ts
import { WorkPaper, exportWorkPaperDocument, serializeWorkPaperDocument } from '@bilig/workpaper'

const workbook = WorkPaper.buildFromSheets({
  Inputs: [
    ['Metric', 'Value'],
    ['Customers', 20],
    ['Average revenue', 1200],
  ],
  Summary: [
    ['Metric', 'Value'],
    ['Revenue', '=Inputs!B2*Inputs!B3'],
  ],
})

const inputs = workbook.getSheetId('Inputs')
const summary = workbook.getSheetId('Summary')
if (inputs === undefined || summary === undefined) {
  throw new Error('Workbook is missing required sheets')
}

workbook.setCellContents({ sheet: inputs, row: 1, col: 1 }, 32)

const revenue = workbook.getCellDisplayValue({ sheet: summary, row: 1, col: 1 })
const saved = serializeWorkPaperDocument(exportWorkPaperDocument(workbook, { includeConfig: true }))

console.log({ revenue, savedBytes: saved.length })
```

## Agent Rules

Agents should stay on contracts and proof, not screenshots.

- Do not assume spreadsheet UI behavior is the public contract.
- Do not hardcode Bilig-owned business models.
- Do not parse workbook ref ids. Use labels and `describeRef`.
- Do not treat planned checks as proof. Proof comes from runtime readbacks or
  verified check results.
- Do not call a mutation successful just because code ran. Inspect the
  `WorkbookRunResult`.
- Prefer package README files, public API docs, runnable examples, and tests
  over guessing from a grid view.

The published package also carries `AGENTS.md` and `SKILL.md` so agents
inspecting `node_modules` can find the local contract. Use the
[agent handoff prompt](docs/headless-workpaper-agent-handbook.md#copy-paste-prompt-for-another-agent)
when handing a workbook task to another agent.

- Node.js `24+`
- Bun

Install and run the local app:

```sh
pnpm install
pnpm dev:web-local
```

```sh
pnpm dev:web        # browser shell
pnpm dev:sync       # monolith runtime
pnpm test           # unit tests
pnpm test:browser   # Playwright tests
pnpm typecheck      # TypeScript project references
pnpm build          # full package build
pnpm run ci         # default CI preflight
```

Focused package checks:

```sh
pnpm --filter @bilig/workbook test
pnpm --filter @bilig/workbook build
pnpm --filter @bilig/workpaper test
pnpm --filter @bilig/workpaper build
```

If you edit protocol or formula inventory sources, regenerate and commit the
generated outputs. CI fails on dirty tracked files.

## Package Footprint

<!-- headless-package-footprint:start -->

Current checked npm footprint for `@bilig/headless@0.47.0`:

- Pack dry run: `739 kB` tarball, `4.51 MB` unpacked, `741` package entries.
- Boundary: the main import is the WorkPaper formula/JSON runtime; XLSX
  import/export stays behind the `@bilig/headless/xlsx` subpath; MCP is the
  `bilig-workpaper-mcp` binary wrapper; reduced workbook reports use the
  `bilig-formula-clinic` binary.
- Cold-start gate: Node imports the main entrypoint, builds a two-sheet
  WorkPaper, and reads `24000` under `1000 ms` without importing
  the XLSX subpath.
- Runtime: Node `>=22.0.0`; Node 22 compatibility is covered by the runtime package workflow.
